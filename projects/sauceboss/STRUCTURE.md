## What This App Does
SauceBoss is a recipe companion that answers "what sauce goes with this?". The user picks a carb (pasta, rice, noodles, bread, potatoes, couscous), optionally filters by ingredients they have or want to avoid, and browses visually rich sauce recipes with step-by-step pie-chart breakdowns of each cooking step. It was originally a React Native app with a local SQLite database; the retrofit moves the data to Supabase and adds a web prototype.

## Data Model
All tables prefixed `sauceboss_` in the shared Supabase project.

- **sauceboss_carbs** — 6 base carbohydrates. Columns: `id` (text PK), `name`, `emoji`, `description`.
- **sauceboss_sauces** — Sauce recipes (22 sauces). Columns: `id`, `name`, `cuisine`, `cuisine_emoji`, `color` (hex), `description`.
- **sauceboss_sauce_carbs** — Many-to-many: which sauces go with which carbs. Columns: `sauce_id`, `carb_id` (composite PK).
- **sauceboss_sauce_steps** — Ordered cooking steps per sauce. Columns: `id` (bigserial), `sauce_id`, `step_order` (int), `title`.
- **sauceboss_step_ingredients** — Ingredients per step. Columns: `id` (bigserial), `step_id`, `name`, `amount` (real), `unit`.

**RPCs** (Supabase PL/pgSQL functions in `db/migrations/002_sauceboss_rpcs.sql`):
- `get_sauceboss_carbs_with_count()` — returns all carbs with sauce count per carb
- `get_sauceboss_sauces_for_carb(p_carb_id text)` — returns fully assembled sauce objects (id, name, cuisine, cuisineEmoji, color, description, compatibleCarbs[], ingredients[], steps[])
- `get_sauceboss_ingredients_for_carb(p_carb_id text)` — returns sorted unique ingredient name list for the filter panel

## API Endpoints
All served by `shared-backend/routes/sauceboss.py` at prefix `/api/v1/sauceboss`.

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | `/api/v1/sauceboss/health` | None | Health check |
| GET | `/api/v1/sauceboss/carbs` | None | All carbs with sauce counts |
| GET | `/api/v1/sauceboss/carbs/{carb_id}/sauces` | None | Fully assembled sauces for a carb |
| GET | `/api/v1/sauceboss/carbs/{carb_id}/ingredients` | None | Unique ingredient names for filter panel |

## Key Business Logic
- A sauce is "available" for the ingredient filter if NONE of its ingredients are in the user's disabled set.
- Unavailable sauces in the React Native app render at 45% opacity and are not tappable.
- The ingredient filter panel shows only ingredients relevant to the currently selected carb.
- Pie chart slices in RecipeScreen are proportional to `amount` converted to teaspoon equivalents (see `src/utils/units.js` if it exists, otherwise `src/components/PieChartLegend.js`).
- `sauces.steps[i].ingredients` is the canonical source for both the per-step pie chart AND the flat `ingredients[]` array (which is derived by deduplication across all steps, preserving first-appearance order).

## Development Setup
```bash
# Shared backend (from vibelab root)
cd shared-backend
python -m venv .venv && source .venv/Scripts/activate  # Windows bash
pip install -r requirements.txt
cp .env.example .env   # fill in SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY
uvicorn main:app --reload --port 8000

# Test backend
curl http://localhost:8000/api/v1/sauceboss/health
curl http://localhost:8000/api/v1/sauceboss/carbs

# Web prototype
open projects/sauceboss/web/index.html  # or serve it:
npx serve projects/sauceboss/web --port 5500

# React Native app
cd projects/sauceboss/app
npm install
echo "EXPO_PUBLIC_API_URL=http://localhost:8000" > .env
npx expo start
```
