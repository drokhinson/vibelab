## What This App Does
SauceBoss is a recipe companion for sauces, dressings, and marinades. The app has three tabs:

- **Sauces tab**: Pick a carb (pasta, rice, noodles, bread, potatoes, couscous) вҶ’ browse sauce recipes with step-by-step pie-chart breakdowns.
- **Dressings tab**: Pick a salad base (romaine, spinach, arugula, mixed greens, kale, cucumber-tomato, shaved beet, quinoa, farro) вҶ’ browse matching salad dressings.
- **Marinades tab**: Pick a protein (chicken, beef, tofu, fish) вҶ’ browse marinades optimized for that protein.

All paths share the same recipe experience: step-by-step cooking cards with pie charts, a serving scaler (1-12 people), unit toggle (Imperial вҶ” Metric), and an ingredient filter panel. The app was originally a React Native app with a local SQLite database; the retrofit moves the data to Supabase and adds a web prototype.

## Data Model
All tables prefixed `sauceboss_` in the shared Supabase project.

- **sauceboss_items** вҖ” Unified selector table for carbs / proteins / salad bases. Type rows have `parent_id IS NULL`; Variant rows (e.g. basmati rice as a prep variant of rice) point at their Type via `parent_id`. Columns: `id` (text PK), `category` ('carb'|'protein'|'salad'), `parent_id` (nullable FKвҶ’items), `name`, `emoji`, `description`, `sort_order`, `cook_time_minutes`, `instructions`, `water_ratio`, `portion_per_person`, `portion_unit`.
- **sauceboss_sauces** вҖ” All recipes (sauces, dressings, marinades). Columns: `id`, `name`, `cuisine`, `cuisine_emoji`, `color` (hex), `description`, `sauce_type` ('sauce'|'dressing'|'marinade').
- **sauceboss_sauce_items** вҖ” Unified junction: sauces вҶ” items. A trigger enforces `sauce_type вҶ” item.category` (sauceвҶ’carb, marinadeвҶ’protein, dressingвҶ’salad) and that links target Type rows only (not Variants). Columns: `sauce_id`, `item_id`.
- **sauceboss_sauce_steps** вҖ” Ordered cooking steps per recipe. Columns: `id` (bigserial), `sauce_id`, `step_order` (int), `title`, `estimated_time` (minutes), `input_from_step` (nullable int).
- **sauceboss_step_ingredients** вҖ” Ingredients per step. Columns: `id` (bigserial), `step_id`, `name`, `amount` (real), `unit`.
- **sauceboss_ingredient_categories** вҖ” Maps ingredient names to filter panel categories.
- **sauceboss_ingredient_substitutions** вҖ” Substitution suggestions shown when an ingredient is marked unavailable.

**RPCs** (see `db/migrations/054_sauceboss_unified_rpcs.sql`):
- `get_sauceboss_initial_load()` вҖ” `{ carbs, proteins, saladBases }` for the home screen (one round-trip).
- `get_sauceboss_item_load(p_item_id text)` вҖ” `{ item, variants, sauces, ingredients }` for any selection screen (one round-trip).
- `get_sauceboss_items_by_category(p_category text)` вҖ” Type rows for one category, with sauce count. Used by `initial_load`.
- `get_sauceboss_sauces_for_item(p_item_id text)` вҖ” Fully assembled sauces linked to an item.
- `get_sauceboss_variants_for_item(p_item_id text)` вҖ” Child rows (parent_id = p_item_id).
- `get_sauceboss_ingredients_for_item(p_item_id text)` вҖ” Sorted unique ingredient names across linked sauces.

## Key Business Logic
- A sauce is "available" for the ingredient filter if NONE of its ingredients are in the user's disabled set.
- Unavailable sauces in the React Native app render at 45% opacity and are not tappable.
- The ingredient filter panel shows only ingredients relevant to the currently selected carb.
- Pie chart slices in RecipeScreen are proportional to `amount` converted to teaspoon equivalents (see `src/utils/units.js` if it exists, otherwise `src/components/PieChartLegend.js`).
- `sauces.steps[i].ingredients` is the canonical source for both the per-step pie chart AND the flat `ingredients[]` array (which is derived by deduplication across all steps, preserving first-appearance order).
